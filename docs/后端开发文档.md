# VideoSum 后端开发文档

> 后端架构设计、核心模块、API接口说明。

---

## 1. 架构概述

```
┌─────────────────────────────────────────────────────────────┐
│                     FastAPI 后端服务                          │
├─────────────────────────────────────────────────────────────┤
│  API层 (backend/api/)                                       │
│  ├── video.py      # 视频处理API                            │
│  ├── config.py     # 配置管理API                            │
│  ├── task.py       # 任务管理API                            │
│  ├── log.py        # 日志API                               │
│  └── plugin.py     # 插件管理API                            │
├─────────────────────────────────────────────────────────────┤
│  核心层 (backend/core/) - 复用现有代码                        │
│  ├── models.py     # 数据模型                              │
│  ├── config_manager.py  # 配置管理                          │
│  ├── downloader.py      # 下载模块                          │
│  ├── scheduler.py       # 核心调度                          │
│  ├── asr/               # ASR引擎                          │
│  ├── llm/               # LLM引擎                          │
│  ├── plugins/           # 插件系统                          │
│  └── extensions/        # 扩展模块                          │
└─────────────────────────────────────────────────────────────┘
```

## 3. 核心模块

### 3.1 数据模型 (`core/models.py`)

**直接复用现有代码**，主要类：

```python
# 枚举
class Language(Enum):        # 支持的语言
class LLMProvider(Enum):     # LLM供应商
class ASRProvider(Enum):     # ASR供应商
class ProcessingStatus(Enum): # 处理状态

# 数据类
class Segment               # ASR转写片段

class TranscriptionResult   # 转写结果

class CostEstimate          # 费用估算
```

### 3.2 配置管理 (`core/config_manager.py`)

**直接复用现有代码**

```python
class ConfigManager:
    def __init__(self, config_dir=None)
    def load(self) -> Dict           # 加载配置
    def save(self)                   # 保存配置
    def get(self, key, default=None) # 获取配置项
    def set(self, key, value)        # 设置配置项
```

**配置文件位置**：`~/.videosummary/config.yaml`

---

### 3.3 核心调度器 (`core/scheduler.py`)

**直接复用现有代码**

```python
class Scheduler:
    def __init__(self, config_manager)
    def create_task(self, input_source) -> str
    def get_task(self, task_id) -> Optional[ProcessingTask]
    def list_tasks(self) -> List[ProcessingTask]
    def delete_task(self, task_id) -> bool
    async def process(self, task_id, input_source, output_language) -> ProcessingTask
    async def qa(self, task_id, question) -> str
```

**线程安全设计**：
- `threading.Lock` 保护 `tasks` 字典
- 原子写入 `state.json`

### 3.6 LLM引擎 (`core/llm/`)

**直接复用现有代码**

```python
# 抽象基类
class LLMEngine(ABC):
    def generate(self, messages, temperature, max_tokens) -> str
    def count_tokens(self, text) -> int
    def is_available(self) -> bool

# 统一客户端（支持9家供应商）
class UnifiedLLMClient(LLMEngine):
    def __init__(self, provider, api_key, model, temperature, max_tokens)
```

**支持的供应商**：
- OpenAI, Anthropic, Google
- DeepSeek, Qwen, Kimi, GLM
- Ollama（本地）

## 4. API接口设计

### 4.1 视频处理API

```python
# POST /api/video/summarize
Request:
{
    "url": "https://www.bilibili.com/video/BV1xx411c7mD",
    "language": "zh"
}

Response:
{
    "task_id": "abc12345",
    "status": "completed",
    "summary": "# 视频总结\n...",
    "processing_time": 45.2
}
```

```python
# POST /api/video/qa
Request:
{
    "task_id": "abc12345",
    "question": "视频讲了什么？"
}

Response:
{
    "answer": "视频主要讲述了...",
    "source_segments": [...]
}
```

```python
# GET /api/video/{task_id}
Response:
{
    "task_id": "abc12345",
    "status": "completed",
    "progress": 100,
    "video_metadata": {...},
    "transcription": {...},
    "summary": {...}
}
```
