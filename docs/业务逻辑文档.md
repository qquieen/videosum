# VideoSum 业务逻辑文档

> 本文档详细描述各模块的业务逻辑，供代码审核和优化参考。

---

## 1. 整体业务流程

```
用户输入(URL/文件)
    │
    ▼
┌─────────────────────────────────────────────────────────────┐
│                    核心调度器 (scheduler.py)                  │
│  1. 判断输入类型（URL/本地文件）                               │
│  2. 下载或加载媒体文件                                         │
│  3. ASR语音转写                                               │
│  4. LLM智能总结                                               │
│  5. 返回结果                                                  │
└─────────────────────────────────────────────────────────────┘
    │
    ├──────────────┬──────────────┬──────────────┐
    ▼              ▼              ▼              ▼
┌─────────┐  ┌─────────┐  ┌─────────┐  ┌─────────┐
│下载模块  │  │ASR引擎  │  │LLM引擎  │  │配置管理  │
└─────────┘  └─────────┘  └─────────┘  └─────────┘
```

## 2. 输入处理模块 (`downloader.py`)

### 2.1 业务逻辑

```
输入: input_source (字符串)

判断类型:
├── 如果是URL → 调用yt-dlp下载音频
└── 如果是本地文件路径 → 直接处理文件

URL判断逻辑:
├── 以 "http://" 开头 → URL
├── 以 "https://" 开头 → URL
├── 以 "www." 开头 → URL
└── 其他 → 尝试作为本地路径

本地文件处理逻辑:
├── 检查文件是否存在
├── 检查文件格式是否支持
├── 如果是视频文件 → 提取音频
├── 如果是音频文件 → 直接使用
└── 获取文件元数据（时长等）
```

### 2.2 数据流

```
输入: "https://www.bilibili.com/video/BV1xx411c7mD"
    ↓
判断: 是URL
    ↓
获取视频元数据 (yt-dlp --dump-json)
    ↓
输出: VideoMetadata {url, title, duration, uploader}
    ↓
下载音频 (yt-dlp -x --audio-format mp3)
    ↓
输出: 音频文件路径
    ↓
返回: (VideoMetadata, Path)
```

### 2.3 关键问题

| 问题 | 当前处理 | 可能的改进 |
|------|----------|------------|
| FFmpeg未安装 | 抛出异常 | 提供安装指引 |
| 文件格式不支持 | 抛出异常 | 列出支持的格式 |
| 下载超时 | 600秒超时 | 可配置超时时间 |
| 会员视频 | 需要cookies | 支持cookies导入 |

## 3. ASR引擎模块 (`asr/`)

### 3.1 业务逻辑

```
输入: audio_path (音频文件路径)
    ↓
加载Whisper模型 (首次加载，后续复用)
    ↓
模型配置:
├── model_size: tiny/base/small/medium/large/large-v3
├── device: auto/cuda/cpu
└── compute_type: float16/int8_float16/int8
    ↓
执行转写 (faster_whisper)
├── beam_size=5 (束搜索宽度)
├── language=None (自动检测语言)
├── vad_filter=True (语音活动检测)
└── vad_parameters: min_silence_duration_ms=500
    ↓
遍历转写结果片段
├── 提取: start, end, text, confidence
└── 记录检测到的语言
    ↓
输出: TranscriptionResult
├── segments: List[Segment]
├── language: str (检测到的语言)
├── duration: float (总时长)
├── provider: ASRProvider
└── processing_time: float (耗时)
```

### 3.2 关键参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `model_size` | `large-v3` | 模型大小，越大越准但越慢 |
| `device` | `auto` | 自动选择GPU/CPU |
| `compute_type` | `float16` | 计算精度 |
| `beam_size` | `5` | 束搜索宽度 |
| `vad_filter` | `True` | 是否启用语音活动检测 |

### 3.3 关键问题

| 问题 | 当前处理 | 可能的改进 |
|------|----------|------------|
| 首次加载慢 | 正常现象 | 显示加载进度 |
| 显存不足 | 自动降级 | 提示用户切换CPU |
| 转写失败 | 抛出异常 | 重试机制 |
| 语言检测错误 | 使用检测结果 | 允许用户指定语言 |

## 4. LLM引擎模块 (`llm/`)

### 4.1 业务逻辑

```
输入: messages (消息列表)
    ↓
选择LLM供应商
├── openai → OpenAI API
├── anthropic → Anthropic API (需适配)
├── google → Google Gemini API (需适配)
├── deepseek → DeepSeek API (OpenAI兼容)
├── qwen → 通义千问 API (OpenAI兼容)
├── kimi → Moonshot API (OpenAI兼容)
├── glm → 智谱 API (OpenAI兼容)
└── ollama → 本地Ollama
    ↓
构建API请求
├── base_url: 供应商API地址
├── api_key: 密钥
├── model: 模型名称
├── messages: 对话历史
├── temperature: 温度参数
└── max_tokens: 最大token数
    ↓
调用API
├── 成功 → 返回生成文本
└── 失败 → 抛出LLMError
    ↓
输出: str (生成的文本)
```

### 4.2 文本总结策略

```
输入: transcription (转写结果)
    ↓
构建文本内容
├── 遍历所有Segment
├── 格式: "[MM:SS] 文本内容"
└── 用换行符连接
    ↓
计算token数量
    ↓
判断是否需要分块:
├── 如果 token数 < 上下文窗口-1000 → 全文总结
└── 如果 token数 >= 上下文窗口-1000 → 分块总结
    ↓
全文总结:
├── 构建System Prompt (包含语言指令)
├── 构建User Prompt (包含完整文本)
└── 调用LLM生成
    ↓
分块总结 (Map-Reduce):
├── 1. 按时间窗口分割文本
├── 2. 每块独立生成摘要
├── 3. 合并所有摘要
└── 4. 再次调用LLM生成最终总结
    ↓
提取关键点
├── 调用LLM提取3-5个要点
└── 解析为列表
    ↓
输出: SummaryResult
├── full_summary: str
├── chunks: List[SummaryChunk]
├── key_points: List[str]
├── total_tokens: int
├── provider: LLMProvider
├── processing_time: float
└── language: Language
```

### 4.3 多语言Prompt模板

## 5. 核心调度器 (`scheduler.py`)

### 5.1 业务逻辑

```
输入: task_id, input_source, output_language
    ↓
创建任务
├── 生成唯一task_id
├── 创建VideoMetadata占位
└── 初始化ProcessingTask
    ↓
执行处理流程:
│
├── 1. 下载/加载 (DOWNLOADING)
│   ├── 进度: 5% → 20%
│   ├── 调用InputHandler.load()
│   └── 更新task.video_metadata
│
├── 2. ASR转写 (TRANSCRIBING)
│   ├── 进度: 20% → 60%
│   ├── 调用asr_engine.transcribe()
│   └── 更新task.transcription
│
├── 3. LLM总结 (SUMMARIZING)
│   ├── 进度: 60% → 95%
│   ├── 调用_summarize()
│   └── 更新task.summary
│
└── 4. 完成 (COMPLETED)
    ├── 进度: 100%
    ├── 计算费用
    └── 返回task
    ↓
异常处理:
├── 任何步骤失败 → 状态设为FAILED
├── 记录错误信息
└── 返回task (带error字段)
```

### 5.2 异步处理模型

```python
# 使用ThreadPoolExecutor执行CPU/IO密集任务
async def process(self, task_id, input_source, output_lang):
    # 下载（IO密集）→ 使用asyncio.to_thread
    metadata, audio_path = await asyncio.to_thread(
        self._input_handler.load, input_source
    )
    
    # ASR转写（CPU密集）→ 使用ThreadPoolExecutor
    transcription = await asyncio.to_thread(
        asr_engine.transcribe, audio_path
    )
    
    # LLM总结（IO密集）→ 使用asyncio.to_thread
    summary = await self._summarize(llm_engine, transcription, output_lang)
```

### 5.3 问答逻辑

```
输入: task_id, question
    ↓
获取任务
├── 检查task是否存在
└── 检查task.transcription是否存在
    ↓
构建上下文
├── 取前20个Segment
├── 格式: "[秒数] 文本"
└── 用换行连接
    ↓
构建Prompt
├── System: 视频问答助手角色设定
├── User: 上下文 + 问题
    ↓
调用LLM
    ↓
输出: str (答案)
```

## 6. 配置管理 (`config_manager.py`)

### 6.1 业务逻辑

```
初始化:
├── 确定配置目录: ~/.videosummary/
├── 确定配置文件: ~/.videosummary/config.yaml
└── 加载配置（如果存在）或创建默认配置
```

## 7. 数据模型 (`models.py`)

### 7.1 枚举定义

```python
class Language(Enum):
    """支持的语言"""
    CHINESE = "zh"      # 中文简体
    ENGLISH = "en"      # 英语
    GERMAN = "de"       # 德语

class LLMProvider(Enum):
    """LLM供应商"""
    OPENAI = "openai"       # OpenAI
    ANTHROPIC = "anthropic" # Anthropic
    GOOGLE = "google"       # Google
    DEEPSEEK = "deepseek"   # DeepSeek
    QWEN = "qwen"           # 通义千问
    KIMI = "kimi"           # Moonshot
    GLM = "glm"             # 智谱
    OLLAMA = "ollama"       # 本地Ollama

class ASRProvider(Enum):
    """ASR供应商"""
    LOCAL_WHISPER = "local_whisper"     # 本地Whisper
    OPENAI_WHISPER = "openai_whisper"   # OpenAI Whisper API
    ALIYUN = "aliyun"                   # 阿里云ASR
    GOOGLE = "google"                   # Google STT

class ProcessingStatus(Enum):
    """处理状态"""
    PENDING = "pending"         # 等待中
    DOWNLOADING = "downloading" # 下载中
    TRANSCRIBING = "transcribing" # 转写中
    SUMMARIZING = "summarizing" # 总结中
    COMPLETED = "completed"     # 已完成
    FAILED = "failed"           # 失败
```

### 7.2 数据类关系

```
VideoMetadata (视频元数据)
├── url: str
├── title: str
├── duration: float
├── uploader: str
├── upload_date: Optional[str]
├── thumbnail: Optional[str]
├── local_path: Optional[str]
└── detected_language: str

Segment (转写片段)
├── start: float          # 开始时间(秒)
├── end: float            # 结束时间(秒)
├── text: str             # 识别文本
├── confidence: float     # 置信度
└── language: str         # 语言

TranscriptionResult (转写结果)
├── segments: List[Segment]
├── language: str
├── duration: float
├── provider: ASRProvider
└── processing_time: float

SummaryChunk (分块摘要)
├── chunk_index: int
├── time_start: float
├── time_end: float
├── summary: str
└── token_count: int

SummaryResult (总结结果)
├── full_summary: str
├── chunks: List[SummaryChunk]
├── key_points: List[str]

├── provider: LLMProvider
├── processing_time: float

QAExchange (问答交互)
├── question: str
├── answer: str
├── source_segments: List[Segment]

└── language: Language

CostEstimate (费用估算)
├── asr_cost: float
├── llm_cost: float
├── total_cost: float
├── token_count: int
├── duration_hours: float
└── currency: str

ProcessingTask (处理任务)
├── task_id: str

├── progress: float
├── current_step: str
├── input_language: Language

├── cost: Optional[CostEstimate]
├── error: Optional[str]
├── created_at: datetime
└── updated_at: datetime
```

## 8. 异常处理策略

### 8.1 异常层级

```python
VideoSumError           # 基础异常
├── DownloadError       # 下载失败
├── ASRError            # 转写失败
├── LLMError            # LLM调用失败
├── ConfigError         # 配置错误
└── PluginError         # 插件错误
```

### 8.2 降级策略

```
ASR降级链:
  阿里云ASR失败
    → OpenAI Whisper API
      → 本地Whisper
        → 报错

LLM降级链:
  DeepSeek API失败
    → 本地Ollama
      → Qwen API
        → 报错
```
